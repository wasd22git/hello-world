# ソフトウェア開発について


### ソフトウェア開発の流れ

ソフトウェアを開発する際、踏襲するべき基本的な流れが存在する。  
まず初めに、顧客の求めるものをくみ取り、どのようなシステムやサービスを作るのかを明確にする要件定義を行う。  
次に、出来上がった要件定義書からスケジュールや人員を考慮して、実際にコードを書いていけるように設計書を作っていく。  
設計が終われば、設計書をもとに製作段階に入っていく。制作段階では、自社で行う場合と外部に委託する場合がある。また、一部の機能のみを外部委託する場合もある。賃金が低い海外の企業に委託する開発手法をオフショア開発と呼び、こういったケースが増えている。  
そして、制作したものが確実に動く保証はないので、テストを行い発見したバグに対してデバッグを行う。すべてをテストすることは難しいので、テストするべき箇所を見極める必要がある。  
最後に、完成したシステムやサービスを稼働させ、運用していく。  
これがソフトウェア開発の大まかな流れである。


### ソフトウェアの評価
ソフトウェアの評価指標は様々あり、複数の指標から分析していくことが重要である。  
簡単な指標がコードの物量である。コードの物量は小さいほど良いとされる。  
また、バグの発生率も評価指標の一つである。完璧にバグをなくすことは難しいケースではどの程度まで発生率を下げるのかを検討すべきである。  
ファンクションポイント法は大まかな要件から開発するシステムの規模を概算する方法である。概算された規模をもとにして、製作段階に入る前に新規開発が必要かどうかを検討することができる。  
そのほかにも、視認性や操作性などの使い勝手の良さも大切な指標となる。  


### ソフトウェアの開発手法

ソフトウェアの開発にはいくつかの手法がある。  
ウォーターフォール開発は要件定義からテストまでの流れを一回で行うシンプルな手法で、進捗の管理がしやすいことがメリットとしてあげられる。  
スパイラルモデルはプロトタイプを作り、評価し、フィードバックを行うプロセスを繰り返す手法である。
反復型開発プロセスではソフトウェアを機能で分割し、それを反復という単位で開発していくことで部分的に完成させていく手法である。
アジャイル開発はソフトウェアを短い期間で開発し、早期にリリースして得られた改善点や変化させたい点を再び開発していく手法である。  
状況に合わせて適切な手法をとることが求められる。  


### スケジュールとタスクの管理

ソフトウェアの開発において、タスクの割り振りとスケジュールの管理は必要となる。ソフトウェアの開発ではWBSで全体のタスクとスケジュールを明確化する。WBSを用いることでやるべきこととその順序関係を整理しやすくなるため、スムーズに作業を行うことができる。


### ソフトウェアの仕組み
ソフトウェアはフロントエンドとバックエンドという2つの部分で構成されている。フロントエンドはユーザー自身が操作する部分である。対して、バックエンドはユーザーが操作しないシステムの部分である。  
インターネットを経由するサービスにはSaaS、PaaS、IaaSの3つがあり、SaaSはソフトウェアを提供するサービス、PaaSは開発環境を提供するサービス、IaaSはサーバーなどのインフラを提供するサービスである。
